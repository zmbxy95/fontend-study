## 强类型
### 什么是强类型？
通常来说强类型语言，变量的类型一经确立，是不允许被修改的。
### 缺点
- 异常需要等到运行时才会被发现
- 函数功能可能发生改变
```javascript
function sum(a, b) {
    return a + b
}

console.log(sum(100, 100)) // 200
console.log(sum(100, '100')) // 100100
```
- 对象索引器的错误用法
```javascript
const obj = {}

obj[true] = 100 // 属性名会自动转换为字符串

console.log(obj['true']);
```
## 弱类型
### 什么是弱类型？
对于弱类型语言来说，变量可以被赋予不同类型的值。
### 优点
- 强类型代码错误可以更早暴漏
- 强类型代码更智能，编码更精确
- 重构更可靠
- 减少了代码层面的不必要的类型推断
